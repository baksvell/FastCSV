# Исправление регрессии для файлов с кавычками

## Проблема

После оптимизаций Medium with quotes стал **3.70x медленнее** (было 1.13x медленнее).

## Причина

Слишком агрессивные оптимизации добавили overhead:
1. Уменьшение порога SIMD с 32 до 16 байт - добавило overhead для маленьких блоков
2. Изменения в резервировании памяти - вызвали больше реаллокаций
3. Усложнение логики создания полей - добавило overhead

## Решение

Откат к более консервативным настройкам:
1. ✅ Вернули порог SIMD к 32 байтам (оптимальный баланс)
2. ✅ Вернули оригинальное резервирование памяти
3. ✅ Упростили логику создания полей
4. ✅ Упростили обработку после закрывающей кавычки

## Статус

✅ Изменения откачены
⏳ Ожидается тестирование
